## 图 1:试菜员在厨房门口(位于 §一,反直觉开场)

- **核心概念**:AI 写代码越快,bug 藏得越深;业务边界没越,但逻辑错了 DDD 也不管;**TDD 才是 CI/CD 里的试菜员**,把每行代码自动试吃再放行。
- **视觉建议**:顾客(用户)在餐桌前等菜,后厨炒菜出锅,厨房门口站着一位戴高帽的试菜员(胸牌"TDD = 试菜员"),每道菜出锅都先尝一口,合格的盖"过"章端出,不合格的退回重做(盖"✗"章)。

## 图 3:覆盖率门槛闸门(位于 §四,怎么做 TDD 之覆盖率与 P0 红线)

- **核心概念**:JaCoCo 强制覆盖率(全集 BRANCH 30% / LINE 70%;**common + domain 各层 ≥ 90%** 行/分支双门槛);< 90% 闸门关闭,代码不能合并。
- **视觉建议**:一座大门,门上挂"common + domain 各层 ≥ 90%(P0 准入)"大牌子;门左侧三个红绿灯(P0/P1/P2 三个等级),P0 不通过时红灯亮(闸门关闭,代码退回重做),P1 亮黄灯,P2 亮绿灯;门右侧站一位 AI 小机器人拿着代码文件准备通过闸门。
- **prompt 草稿**:暖色调奶油纸背景,黑色草图线条,柔和粉彩色块(主色橙+绿+蓝+红)。画面中央一座大门,门上挂着一块大牌子"覆盖率门槛 = common + domain ≥ 90%(P0 准入)";门左侧三个红绿灯并排(P0 红灯 / P1 黄灯 / P2 绿灯),P0 红灯亮起,旁边标"闸门关闭 → 代码退回重做";门右侧 AI 小机器人(橙色)抱着一个写着"@Test"的代码文件,表情焦急等闸门。底部小字"JaCoCo 强制:全集 BRANCH 30% / LINE 70%,common + domain 各层 ≥ 90% 行/分支双门槛"。右下角 ArchAIHarness 水印。

## 图 4:AI 时代 TDD 三大纪律(位于 §五)

- **核心概念**:三大纪律——① 覆盖率门槛 AI 不能自主决定(人定 90%,AI 不能改);② P0 红线不能绕(绕一次整个工程的纪律就崩了);③ 测试不能"自己测自己"(禁止 Mock 实体,必须真造真测)。
- **视觉建议**:三块警示牌竖排,每块警示牌对应一条纪律,用后厨"厨房公告板"的形式展示;背景是后厨墙上钉着三块告示牌(对应"严禁"、"注意"、"警告"等)。
- **prompt 草稿**:暖色调奶油纸背景,黑色草图线条,柔和粉彩色块(主色橙+红+黄)。画面中央厨房墙上钉着三块警示牌(从左到右):① 第一块"覆盖率门槛人定 AI 不改"——画面里一位主厨(架构师)伸手按 90% 数字,旁边 AI 小机器人想改被推开;② 第二块"P0 红线不能绕"——一面红墙(P0 红线),AI 试图绕路被拦住;③ 第三块"禁止 Mock 实体"——画面里一位试菜员举着"禁止假食材"标语。每块警示牌底部小字标注"AI 时代 TDD 三大纪律 / 必须遵守 / 不可突破"。右下角 ArchAIHarness 水印。

## 图 5:CI/CD 流水线拦截图(位于 §六,怎么集成 CI/CD 流水线)

- **核心概念**:AI 写代码 → 单元测试 → 集成测试 → 覆盖率闸门 → P0 红线检查 → 通过(放行)/ 不通过(退回重做)——一条横向流水线,每道闸门都会自动拦截不合格的代码。
- **视觉建议**:横向一条流水线,左侧 AI 小机器人写代码(进入流水线),中间 4 道自动闸门(单元测试 / 集成测试 / 覆盖率 / P0 红线),右侧两条分叉——通过(放行,代码上生产) / 不通过(退回 AI 重做);不通过的代码用红色叉号标记退回。
- **prompt 草稿**:暖色调奶油纸背景,黑色草图线条,柔和粉彩色块(主色橙+绿+蓝+红)。画面横向一条流水线,左侧一位 AI 小机器人抱着代码文件进入流水线;中间 4 道自动闸门(从左到右):① "单元测试" ② "集成测试" ③ "覆盖率闸门" ④ "P0 红线检查"——每道闸门都是铁门+红绿灯;右侧分叉——通过(绿色箭头,代码上生产) / 不通过(红色叉号,代码退回 AI 小机器人重做)。流水线底部一长串命令行框(mvn test / mvn verify / JaCoCo check 等)。底部小字"AI 时代 CI/CD = 试菜员常驻厨房门口"。右下角 ArchAIHarness 水印。
