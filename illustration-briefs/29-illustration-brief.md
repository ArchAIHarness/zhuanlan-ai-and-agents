## 图 3:厨师长管一道菜(位于 §四,怎么做 DDD 之战术设计)

- **核心概念**:聚合根像厨师长 —— 一道菜的所有步骤都得经过他(选材→切→炒→装盘),谁都不能擅自越界。
- **视觉建议**:一道菜从左到右的全流程(食材→切配→炒锅→装盘→端出),中央站一位戴高帽的厨师长(胸牌"聚合根"),手举"IN/OUT"双向箭头。所有工种区的人(切配、炒锅、装盘)都得经过他签字才能交接,最后端菜出厨房门。
- **prompt 草稿**:暖色调奶油纸背景,黑色草图线条,柔和粉彩色块。画面中央一道菜从左到右的工位流水线:① 食材篮 ② 切配台 ③ 炒锅 ④ 装盘台 ⑤ 出菜口。每个工位上方挂一块小招牌标注工种。每个工位面前站着一位戴厨师帽的小厨师,工位与工位之间有一条传送带/手递手箭头,箭头在切配→炒锅→装盘→出菜这四个节点都汇聚到中央的一个厨师长身上——他站画面中央,戴高帽、系围裙、双手举起"IN/OUT"双向牌子,胸牌手写体"聚合根=厨师长"。背景墙上挂着"菜单(统一语言)"和"工艺标准"。底部小字"一道菜的所有步骤都得经过厨师长,谁都不能擅自越界"。右下角 ArchAIHarness 水印。

## Markdown 排版规范(撰稿员必守,2026-07-01 用户校准)

撰稿输出严格按以下 Markdown 风格,从已读 28 篇里抽出的格式特征:

### 标题层级
- `# 文章标题`(第一行 H1,标题里**没有**"看懂 X"的开头)
- `## 一、xxx`(中文数字 + 顿号,节标题)
- `## 七、xxx` 或省略(末节不需要"结语"标签,直接接最后一节标题)
- `### xxx`(三级标题,子节)
- 不使用 H4

### 段落空行
- **每段独立,段间空行**(关键纪律)
- 段落长度:**1-3 句为主**,关键金句独立成段,最长不超过 5 句
- 不连续写大段(避免"段落墙")

### 强调
- `**加粗**` 用于核心金句、关键概念词
- **金句独立成段**(标志性写法):
  ```
  **AI 不是不需要规矩,AI 是更需要规矩——因为它能放大所有好事,也能放大所有坏事。**
  ```

### 引用
- `> 引用文字` 用于对话、示例、反问
- 引用块与正文间空行
  ```
  > 你跟 AI 说:"帮我写个文案。"
  ```

### 列表
- `- xxx` 列表项;列表项之间空行
- 不使用 `1. xxx` 数字列表(避免和节标题冲突)

### 代码
- `` `代码` `` 用于技术术语、API 名、文件路径、配置项
- 不使用 ``` 代码块 ``` 包裹散文

### 图片
- `![alt](path)` 放在段落后,**图片本身占一行**,前后空行
- 图片路径格式:`assets/29_<文章名>/NN-<slug>.png`
- 图片位置用占位符 `@@@IMG_N@@@`(共 5 张:1-5 按出现顺序)

### 章节分隔
- 节与节之间用 `---` 分隔
- `---` 上下各空一行

### 破折号
- **中文破折号** `——`(不用英文 `—` 或 `--`)
- 用于解释、转折、递进

### 引号
- **中文双引号** `""`(不用英文 `""`)
- 标题里的核心词用双引号包住:`从"会聊天"到"能干活"`

### 结尾
- **不写** `## 结语` 字样
- 直接进入"## 七、xxx"(最后一节标题)
- 然后接 `---` + `### 关于 ArchAIHarness` + 标准模板
