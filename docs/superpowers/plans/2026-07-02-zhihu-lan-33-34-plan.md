# 33-34 期 · 行道阶段第一组（AI 创课助手） 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 落盘两篇专栏文章——第 33 篇《业务心智篇》(奇数理论篇,有下一篇预告)+ 第 34 篇《工程落地篇》(双数实践篇,无预告,金句收束)。业务素材:老师用 AI 创建课程的 ToB 真实业务(基于 FF_快速验证 单元 AI 创课助手项目,跨单元授权引用)。全套流程跑完保存到三平台(知乎/CSDN/掘金)草稿就绪。

**Architecture:** 5 角色团队(content-researcher 调研 → content-writer 撰稿 → content-illustrator 配图 → content-auditor 审核 → content-publisher 发布)。**合并调研**(33+34 一份调研报告)→ **独立撰稿**(33 上篇 + 34 下篇 各一份正文)→ **独立配图**(每篇 5+1 手绘教育插图)→ **三平台独立上稿**(用户手动发布)。所有产物沉淀到本仓库既有约定位置。

**Tech Stack:** Git + Markdown + 手绘教育插图(baoyu-image-gen via baoyu-cover-image + baoyu-article-illustrator skill)+ Playwright(发布自动化)+ 本仓库 content-* 协作纪律(`AGENTS.md`)。

## Global Constraints

- **跨单元纪律(必守,来源: `docs/superpowers/specs/2026-07-02-zhihu-lan-33-34-design.md` §一.1.3)**:
  - 可引用 ADD 教学设计模型(公开)、JSON Schema(公开)、行业实践(教育技术公开事实)
  - 可借鉴 FF 单元的课程知识体系 JSON 输出结构(脱敏后讲设计思想)
  - **不复述** FF 单元 435 行设计文档原文、内部 skill 调用链路
  - **不调度** FF 单元 `course-architect-add` skill 或其他 FF 资源
  - **不强加** 本单元内容创作方法论
  - 专栏口径:"我做的某个 ToB 课程设计项目"——不复述、不点名

- **写作纪律(必守,来源: `AGENTS.md` §5)**:中文/短句/术语首次出现给直觉解释/6-8 节/核心比喻 1 个/去 AI 味清单/开头四步走 + 结尾四步走。

- **品牌串联(必守,来源: `AGENTS.md` §6)**:每篇至少一处内嵌钩子自然引到 ArchAIHarness 主张 + 文末统一出口段(§6.2 模板)+ 不暴露私有 `.opencode/` 编排(§5.7)。

- **维护约定(必守,来源: `AGENTS.md` §七)**:§7.0 完稿五要素全数就位才算完成 + §7.4 发布门禁(发稿前主编必须用 question 工具逐平台确认,任何平台不得合并、不得绕过)+ §7.3 发布链接自动归档。

- **连载节奏(必守,来源: `AGENTS.md` §5.2)**:**33 篇(奇=心智篇)末尾必须写下一篇预告**指向 34 篇;**34 篇(偶=落地篇)末尾不写预告**,直接以金句收束(参考 28/30/32 实践篇的收束风格)。

- **承担撰稿约束(必守,33-34 期 spec)**:
  - **33 篇**:核心比喻"标准菜谱"系列 + 中心金句"功能不是工程师主观设计,只是把老师原生需求提纯、规范化"
  - **34 篇**:核心比喻"标准菜谱厨房"系列(沿用 27/28 厨房三件套 + 33 菜谱延展)+ 中心金句"AI 不懂 ADD,但 AI 能把 ADD 跑得比人更稳"
  - 两篇**禁止重复** 27/28/29/30/31/32 用过的核心比喻(乐谱/作曲家/赛道/国家边界/轨道)
  - 33 篇含 **3-4 轮追问上限纪律** 段(为 34 篇 Spec 端硬约束埋钩子)
  - 34 篇必含 **资产沉淀复盘** 段(呼应用户规划"价值复利、脱离时间换钱"立意)

- **目录与命名(必守,来源: `AGENTS.md` §4.4)**:专栏正文 `NN_<标题>.md`;配图 `assets/NN_<标题>/`;渠道包按平台独立存放;插画简报 `illustration-briefs/33-illustration-brief.md` / `illustration-briefs/34-illustration-brief.md`;调研报告 `research/33-34-research-report.md`;审核报告 `audit/33-audit-report.md` / `audit/34-audit-report.md`。
