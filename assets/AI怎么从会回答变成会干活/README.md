# Tool、Skill、MCP 文章配图规划

对应文章：[`AI 怎么从“会回答”变成“会干活”？看懂 Tool、Skill 和 MCP`](../../AI怎么从会回答变成会干活.md)

本文配图目标：帮助普通用户直观理解 AI 从“会回答”走向“会干活”的外部能力体系，同时让行业从业者看到 Tool、Skill、MCP 的能力供给价值。

## 统一视觉口径

- **风格**：延续本专栏科普型技术插画风格，清晰、克制、专业，不做夸张赛博视觉。
- **色彩**：深色背景 + 蓝紫/青绿色科技辅助色，局部用暖色强调风险和边界。
- **文字**：图片内文字尽量短，优先使用关键词，不放长句。
- **构图**：每张图只解释一个核心概念，避免信息过载。
- **边界**：不出现真实客户、内部系统、真实账号、密钥、品牌后台截图。

## 建议配图清单

### 01-ai-answer-vs-action.png

- **位置**：文章开头或第一节后。
- **作用**：解释“会回答”和“会干活”的区别。
- **画面**：左侧是 AI 对话气泡，只能输出建议；右侧是 Agent 执行链路，包含读文件、查网页、改文档、保存结果等动作。
- **关键词**：回答 / 执行 / 外部能力。
- **重点**：让读者一眼看懂：模型生成答案，Agent 借助外部能力推进任务。

### 02-tool-as-hands.png

- **位置**：第二节“Tool：AI 的手”后。
- **作用**：解释 Tool 决定 AI 能操作什么。
- **画面**：中心是 AI 大脑或 Agent 核心，向外伸出多只“工具手臂”，分别连接网页、文件、数据库、命令行、API、日程等。
- **关键词**：Search / File / API / DB / Command。
- **重点**：突出 Tool 是具体操作能力，同时标注只读、写入、危险动作三种权限层级。

### 03-skill-as-method.png

- **位置**：第三节“Skill：AI 的做事方法”后。
- **作用**：解释 Skill 不是插件，而是可复用任务方法。
- **画面**：一条任务流程卡片，从“明确目标”到“读取资料”到“生成草稿”到“检查风险”到“输出结果”。旁边放几个可复用 Skill 卡片：写作、代码审查、日报、会议纪要。
- **关键词**：流程 / 经验 / 验收 / 复用。
- **重点**：让读者理解 Tool 解决能不能做，Skill 解决怎么做。

### 04-mcp-standard-socket.png

- **位置**：第四节“MCP：AI 连接外部能力的标准插座”后。
- **作用**：解释 MCP 是标准连接方式。
- **画面**：Agent 位于中心，MCP 像统一插座或接口层，外部连接本地文件、浏览器、GitHub、数据库、知识库、办公系统。
- **关键词**：MCP / Standard Interface / Tools / Data Sources。
- **重点**：强调 MCP 解决连接问题，不代表自动安全可靠。

### 05-where-users-get-capabilities.png

- **位置**：第五节“普通人从哪里获得这些能力？”后。
- **作用**：给普通用户建立获取渠道认知。
- **画面**：四个入口卡片：AI 产品内置能力、插件/扩展/Skills、MCP 服务、封装好的 Agent/工作流。
- **关键词**：内置能力 / 插件 / MCP / 工作流。
- **重点**：不画具体产品操作界面，只画能力来源地图。

### 06-capability-supply-for-builders.png

- **位置**：第六节“从业者的机会”后。
- **作用**：解释行业从业者如何供给能力。
- **画面**：左侧是业务系统、行业经验、数据源、专业流程；中间经过 Tool / Skill / MCP 三个封装层；右侧输出给 Agent 生态。
- **关键词**：业务能力 / Tool 供给 / Skill 供给 / MCP 服务 / Agent 生态。
- **重点**：突出“未来很多系统不只是给人用，也要给 Agent 用”。

### 07-boundary-audit-loop.png

- **位置**：第七节“能力越强，越需要边界和审计”后。
- **作用**：承接品牌主张，解释权限与审计闭环。
- **画面**：人定义目标和边界，Agent 执行 Tool/Skill/MCP，系统记录日志、验证结果、触发人工确认，形成闭环。
- **关键词**：边界 / 权限 / 确认 / 日志 / 审计。
- **重点**：突出“人立法，AI 执行，体系审计”。
