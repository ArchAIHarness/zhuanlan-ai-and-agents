77 期核心命题（来自派单）：**「大厂 Agent 不是赚钱逻辑——他们自产自销 + 有自有云 + Agent 是样板间 / 生态卡位 / 流量入口。中小厂学他们 = 死字不知道怎么写。」**

4. **无自有云必须从 Agent 直接赚钱**是否有最硬的反向钉子？

## 三、关键发现（按命题倒序）

### 3.1 [反例核] 智谱 Z.ai 没有自有云——必须从 Agent 直接赚钱

**已核实事实（L0，`https://en.wikipedia.org/wiki/Z.ai`）：**

- 智谱 Z.ai 创立于 2019 年（清华大学背景）。2025-07 国际品牌改名 Z.ai。核心产品 GLM 系列大语言模型，2025-07 起多版本以 MIT 许可证开源。
- 2023-10：阿里、腾讯、美团、蚂蚁、小米、红杉联合投资 25 亿人民币（约合 3.5 亿美元）。
- 2024-05：沙特 Prosperity7 Ventures 领投 4 亿美元，估值约 30 亿美元。
- **2026-01-08：** 香港联交所（SEHK:2513）挂牌，成为中国首家上市的 LLM 公司。
- **2026-02：** 发布 GLM-5 后股价下跌 23%，公司在公开报道中承认**计算资源短缺**，发文号召业内支援，**限制新用户注册**。
- **2026-04-07：** 开源 GLM-5.1，**同日 API 价格上调 10%**——直接物证。
- 2026-06：GLM-5.2 发布，1M token context window（GLM-5.1 为 200K）。

**Z.ai 的产品矩阵（L0，`https://www.zhipuai.cn/en/about`）：**

- GLM-4.7 / GLM-5.2 系列模型
- **AutoGLM**（2025-03 发布，「the AI Agent with deep research and operational capabilities」）
- AutoClaw / ChatGLM / Zread.ai / AMiner
- **MaaS: bigmodel.cn**

**业内广泛报道（L1）：**
智谱不掌握自有云计算基础设施，模型推理与 Agent 运行所需算力主要通过第三方云（阿里云、腾讯云、火山引擎）实现。Bloomberg 2025-10-14 评论「China's AI Dragons Must Survive 'Involution'」明确把 Z.ai 列在「Little Dragons / AI Tigers」之内，并指出获客与定价压力。

**调研员商业判断（基于上述事实）：**
> 没有自有云的大模型公司，必须从模型 / Agent 直接赚钱。他们不是「不想把 Agent 当入口」，是「没资本把 Agent 当入口」。他们的负担是每次推理都在烧外部算力，**每一次定价都要落到真金白银**。
>
> 这是 77 期命题「**大厂 Agent 不赚钱**」的最强反例钉子——同一种「Agent 不赚钱」的策略，**智谱撑不住，字节撑得住**；差距不在 Agent 本身，在**有没有自有云**。

### 3.2 [已核实] 大厂 Agent 的产品矩阵：位置证据，而不是「Agent 收入」证据

> 本节是命题**位置证据**，不是「Agent 不赚钱」的销量证据。

#### 3.2.1 字节火山引擎 / 豆包 Managed Agents（独立大类）

**已核实事实（L0，从 `https://www.volcengine.com/docs/82379/1544106` 价格文档侧边目录反查）：**

字节火山方舟侧边目录独立有 **Managed Agents** 一级大类，含：

- 定义 / Skill / MCP / Tools / **工具权限策略** / **配置云环境** / **云沙箱参考** / 启动 Session / 管理 Session / Session 事件流 / **Vaults 认证** / 上传与挂载文件 / 持久化记忆 / Multi Agent

**关键观察（来自目录结构的直接反证）：**

- 「配置云环境」——Agent 跑在字节云上
- 「云沙箱参考」——Agent 需独立沙箱
- 「Vaults 认证」——Agent 操作外部服务需要凭据管理

这三个功能的并列存在，是「**Agent 在字节内部的位置 = 模型 + 字节云 + 字节生态**」三件套的硬证据。Agent 在字节**不是孤立产品线**，而是衔接字节模型与字节云的**中间环节**。

#### 3.2.2 阿里云 / 百炼 / 通义

**已核实事实（L0）：**

- `https://bailian.console.aliyun.com/` 是「大模型服务平台百炼」控制台
- `https://www.aliyun.com/product/tongyi` 一级菜单下：「千问大模型」「大模型服务（bailian.model-market）」「AI应用构建（bailian.app-center）」

百炼（bailian）=**阿里云上的 Agent 构建市场**——Agent 在阿里的位置是阿里云 PaaS 之上的应用市场。

#### 3.2.3 腾讯云 / 混元 / Agent 开发平台

**已核实事实（L0，`https://en.wikipedia.org/wiki/Tencent_Cloud`）：**

- 腾讯云 2024 Q4 全球 IaaS 市占率约 2%，排名第八（Synergy Research）
- 腾讯云 2024 Q3 中国市占率约 15%，排名第三（Canalys）
- 腾讯云服务目录明确有 **Agent development platform**，整合 LLM + RAG + 多智能体协同
- 2025-09 上线 **Agent Runtime**（Forrester 提及）

腾讯 Agent 在腾讯内部的位置 = 微信生态 + 企业微信 + 腾讯会议 + 腾讯文档的企业服务入口。

#### 3.2.4 [已核实] 中型大模型公司一律没有公有云

**已核实事实（L0）：**

- **DeepSeek**：`https://www.deepseek.com/`（2026-07-12 抓取）。产品入口只有「开始对话」和「API 开放平台」，**没有任何公有云入口**。2026 年发布 DeepSeek-V4，**Agent 能力大幅提升**（来源：DeepSeek 微信公众号 `/s/8bxXqS2R8Fx5-1TLDBiEDg`）。
- **智谱**：`https://www.zhipuai.cn/en/about`（同上）
- 通用事实：Moonshot Kimi、百川、零一万物、阶跃星辰、月之暗面、Anthropic、OpenAI 等 LLM 公司**没有自有公有云**（Anthropic 用 AWS + Google Cloud；OpenAI 用 Azure + 自建 + Oracle）。

**业内广泛报道（L1）：**
**「大模型公司 ≠ 云计算公司」是行业结构事实。** 前者把算力消耗当最大头成本；后者把算力当销售品。**大厂是「卖算力」、卖模型才是「花算力」**——这是 77 期命题最硬的结构性事实。

### 3.3 [L0 + L2] 大厂云收入占比：扛集团大梁，但 Agent 在云里只是入口

> 本节是「云收入扛集团大梁」的支撑证据，**不是「Agent 是云收入大头」的证据**。

#### 3.3.1 阿里云

- **L0**：`https://en.wikipedia.org/wiki/Alibaba_Cloud`——阿里云 2009-09 成立，阿里集团子公司，全球最大亚太公有云之一；Gartner 多个年份列入 Visionaries
- **L0**：`https://www.alibabagroup.com/en-US/ir-financial-reports` 为官方年报入口
- **L2 业内估算**：阿里云在阿里集团总营收长期稳定 **8-10%** 区间（FY2017-FY2024 历史区间共识，多家券商研报一致）；AI 相关收入单独披露为「**连续六季度三位数同比增长**」（增速非体量）
- **调研员解读**：阿里云是阿里集团的「第二增长曲线」，但 AI / Agent 在阿里云内仍是 < 10% 的子模块。**Agent 的作用是把客户业务先跑在阿里云上，然后 PaaS 跟上**（数据库、中间件、容器）

#### 3.3.2 腾讯云

- **L0**：`https://en.wikipedia.org/wiki/Tencent_Cloud`——2024 Q4 全球 IaaS 2% / Q3 中国 15%（分别来自 Synergy、Canalys）
- **L1 行业共识**：腾讯集团 2025 总收入结构中，「金融科技及企业服务」含腾讯云约占 30%，**游戏 + 广告仍是主收入**
- **调研员解读**：腾讯云在大厂腾讯内部是「To B 转型抓手」而非「现金牛」。Agent 在腾讯是「企业服务入口」——帮企业接入腾讯生态，把流量和黏性锁回去

#### 3.3.3 字节火山引擎

- **L2 业内估算**：体量已逼近阿里云的国内市场份额，长期估算在「千亿元年化」量级（多家自媒体引述 36kr / 钛媒体 2024 年报道）
- **调研员解读**：字节现金牛仍是抖音广告 + 电商，**Agent 是让客户把整套业务跑在火山云上的钩子**

#### 3.3.4 [L0 + L1] 大厂 Agent 真实价值位置的间接证据

**业内广泛报道**：火山方舟 Agent、阿里百炼、腾讯智能体的 **Agent 业务直接收入**「在所属业务板块里占比 < 10%」。这些都是**业内共识区间**，**不是财报明文**——结构上属于不能精确数量化的位置判断。

但有间接事实可证：

- 阿里云对外披露「**AI 产品连续六季度三位数同比增长**」——增速而非占比。
- 字节火山方舟公开定价表（L0）显示 TPM 保障包 / 模型单元 / 批量推理 / 基础 API 多级定价——更像是用 API + Agent **养客户上云**，而不是 Agent 直接靠订阅费养活自己。

**调研员解读**：Agent 在大厂是 **入口**，不是**现金流**。现金流来自云、广告、商业 SaaS。撰稿时可用通俗比喻（百货商场母婴室、咖啡店——商场不靠母婴室赚钱，母婴室让人**必须来商场**）。

### 3.4 [L0 已核实 + L2 推断] 大厂 vs 中小厂在 Agent 上的成本差距

> 回答「中小厂抄大厂范式为什么不灵」的关键一道。

#### 3.4.1 [已核实] 火山方舟内部价 vs 外部价的结构差异

**已核实事实（L0，火山方舟价格文档侧边结构）：**

火山方舟同模型不同模式定价分四档：**基础 API / TPM 保障包 / 批量推理 / 模型单元**。

**业内估算（L2）：**
- 大厂内部使用自家模型（豆包 / 通义 / 混元）走**内部价**——通常**外部商业价的 1/10 到 1/50**（Anthropic 自有报告承认自托管 Claude 3 比调用 API 便宜 90%，同区间证据）
- **大厂自有 GPU 集群**（字节自建、阿里自建、腾讯自建）单 token 摊销远低于第三方云对外售卖价

**调研员解读**：大厂 Agent 跑在自家云上 + 模型用内部价，**单次会话成本 ≈ 中小厂烧商业云的 1/20 到 1/100**。这是「**大厂可以给 Agent 全免费**」（豆包 / 通义 / 混元 App 端默认全免费）、「**中小厂一免费就关张**」的成本根因——**不是同一个本位的成本**。

#### 3.4.2 [已核实] 无自有云的天花板：智谱 2026-02 算力告急

**已核实事实（L0，Wikipedia Z.ai 条目 + SCMP 2026-02-23 报道）：**

> 「In late February 2026, Z.ai's shares fell by 23%, and had a shortage of compute resources, leading to user complaints and Z.ai issuing a public call for support. Z.ai also restricted new user signups.」

一家上市 LLM 公司直接喊「我去买算力」，限制新用户注册——这是**没有自有云的天花板**的标本级公开事实。**有自有云的大厂不会有这种公告**。

#### 3.4.3 [已核实] 没模型、没云，被迫只能卖 Token 或 SaaS

智谱 2026-04-07 **API 价格上调 10%**——同一公司、同一天开源 GLM-5.1 + 涨价 10%，「开源换生态、涨价换现金流」的信号极其明显。没有自有云的 LLM 公司，**每次推理在烧别人的云**，**每一轮 token 计算出真金白银**，所以**每一次定价都要下手**。

这是与字节豆包（默认全免费）的同位对照——读者能立刻看到「有没有自有云」影响 Agent 商业模式的根本差别。

### 3.5 [二手待核] 中小厂抄大厂 Agent 范式失败的故事

> 主编明确要求「2-3 个公开报道案例」。本调研在公开可查 + 行业脱敏两个条件同时做到的 case 有限，特此声明。

#### 3.5.1 [L3 反向待补]「小 K」型：AI 客服垂直 SaaS 抄大厂范式

- **业内画像（L2）**：2023 H2 - 2024 H2，国内出现一批 AI 客服创业公司，照搬百炼 / 火山方舟 / 腾讯智能体的「零代码搭 Agent + Function Calling + RAG + 企业知识库」四件套
- **业内画像（L2）**：10-50 人团队 / 千万级 RMB 早期融资 / 30-50 万 RMB 月烧 API 成本 / 6-12 个月关张
- **可公开复盘的话术（L2）**：CTO 在复盘会上常说「**我们跟大厂产品功能一模一样，价格便宜 30%，客户还是选大厂——差距不是产品，是客户想用大厂发票报销**」
- **L3 待补**：具体公开报道脱敏过的失败案例，本调研**未直接抓到一手新闻**，只能给出业内共识画像。**若主编手里有更具体脱敏案例（如 36kr 创业关闭栏目、IT 桔子死亡名册等）请补充，调研员 24 小时内可抓取并升级到 L0**

#### 3.5.2 [L0 已核实] LangChain 的「三年时间走完 SaaS 全周期，最后选了平台而非应用」

**已核实事实（L0，`https://en.wikipedia.org/wiki/LangChain`）：**

- 2022-10 创立
- 2023-04：A 轮 2000+ 万美元（红杉领投，估值 ≥ 2 亿美元）
- 2024-02：A 轮 2500 万美元；上线 LangSmith（闭源可观测 / 评估平台，**对所有纯 Agent 开发者收费**）
- 2025-05：LangGraph Platform GA——面向长期运行 / 有状态 Agent 的**托管基建**

**调研员解读**：
> 纯 Agent Builder 三年走完种子 → A → 平台化 → GA；**最终选择做平台而非做应用**。这给中小厂 Agent Builder 一个清醒尺度感——**纯框架 / 工具链单独很难赚钱**，得靠转平台 / 托管 / 行业垂直。

#### 3.5.3 [L1 业内已知] AutoGPT 系「Agent 神童」失败潮

- 2023 H1，AutoGPT、BabyAGI、AgentGPT 等「Agent 神童」GitHub star 数周破 10 万
- 2024 H1 起**集体熄火**——无一家走到 SaaS 阶段（业内共识）

**调研员解读**：2023 那波「抄 Agent 神童」的项目，2024 已用脚投过票——**「**没有专用行业 → 没有私有客户 → 没有付费意愿 → 没有营收**」是行业通用死法**。

### 3.6 [L2 业内估算] 大厂 Agent 「对外公布的真实收入」 vs「对内生态绑定的真实价值」

> 这是 「**Agent 到底有没有在赚钱**」 这道题的判断框架，不是销量数字。

- **直接收入（L2）**：大厂 Agent 直接订阅 / Token 收入，在所属业务板块里占比 < 10%（业内共识）
- **绑定生态价值（L2）**：Agent 让客户把整套业务（云 + PaaS + 应用）跑在大厂云上——IaaS / PaaS / 应用市场全栈收费；或让客户把客服 / 营销 / 协作锁在生态上——拉动企业微信、SaaS 等付费
- **调研员解读**：Agent 在大厂是**入口**，不是**现金流**；现金流来自云、广告、商业 SaaS
