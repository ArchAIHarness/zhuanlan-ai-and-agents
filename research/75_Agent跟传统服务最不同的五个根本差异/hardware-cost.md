## 3. 完整算账(中小 AI 客服初创"小 K"场景)

假设:1000 用户 × 每人每天 10 轮对话 × 半年 累计,所有对话走 OpenAI GPT-5.5 标准 API。

### 3.1 Token 算账

- 单轮 input + output 平均 4000 tokens
- 1000 用户 × 10 轮 × 4000 tokens × 180 天 = **7.2 B tokens / 年**
- Input:Output 比例 2:1(典型 agent)
- 单价取 GPT-5.5 标准(input $5/M, output $30/M)
- Token 成本 ≈ 7.2 B tokens × (2/3 × $5/M + 1/3 × $30/M)
- ≈ 7200 M tokens × ($3.33 + $10.00) / M
- ≈ 7200 × $13.33
- ≈ **$96,000 / 年**(≈ ¥69 万)

### 3.2 硬件算账

- 1000 个 1C1G VM,经济型突发性能均值 ¥450/年
- **¥450,000 / 年**

### 3.3 额外成本(估)

- 数据库 / Redis / 对象存储 / 监控日志 / 带宽 ≈ **¥100,000-200,000 / 年**

### 3.4 总成本(年度)

| 项目 | 金额 (人民币) |
|---|---|
| Token | ~¥690,000 |
| 硬件(1C1G × 1000) | ~¥450,000 |
| 数据库 + 监控 + 带宽 + 对象存储 | ~¥150,000 |
| **合计** | **~¥1,290,000(129 万)** |

### 3.5 收入对比(中国 SaaS 客户付费意愿)

| 用户付费意愿区间 | 月费 / 用户 / 年 | 1000 用户年收入 |
|---|---|---|
| 微中小 SaaS(占市场 60%) | 50-300 元/月 | 60-360 万 |
| 中型 SaaS(占市场 30%) | 200-800 元/月 | 240-960 万 |
| 大型 SaaS(占市场 10%) | 800+ 元/月 | 960+ 万 |

> **结论**:即使按最乐观的中型 SaaS 客单价(¥800/月)算,1000 用户年收入 **960 万** —— 减去总成本 **129 万** —— **毛利 ~870 万**。但:**这只是纸面毛利率**;如果 LLM 用更强模型(Sonnet 5 / Claude Opus)或长上下文(GPT-5.5 Pro),账单要再翻 2-10 倍;如果算上研发摊销(agent 开发的人工)、客户成功、销售费用,**净利率会被打穿**。

**L2**(算法估算,基于 L4 厂商单价 + 行业经验)。建议主编在文中标注"以本调研报告当期架构和单价为基准,实际项目预算上浮 30-50% 较为稳妥"。

## 4. 1C1G 不是最终选型 — 但它是算账起点

绝大多数 agent 应用的真实部署不是 1C1G,而是 4C8G / 8C16G 甚至 GPU 实例。本期选 1C1G 是因为它算账最直观、最有冲击感。

如果换成 4C8G:
- 阿里云 4C8G 标准 ≈ ¥300/月
- 1000 实例 × 4C8G / 年 ≈ **¥3,600,000(360 万)**
- 是 1C1G 的 10 倍

| 配置 | 单实例月费 | 1000 实例年费 |
|---|---|---|
| 1C1G(突发) | ¥30-50 | 36-60 万 |
| 2C4G(标准) | ¥100-150 | 120-180 万 |
| 4C8G(标准) | ¥250-350 | 300-420 万 |
| 8C16G(标准) | ¥500-700 | 600-840 万 |
| 1×T4 GPU(8C16G) | ¥1500-2500 | 1800-3000 万 |

> 这张表是为了让读者直观感受到"1000 个 agent 实例"的真实成本。**1C1G 已经是最乐观的底线值**。
